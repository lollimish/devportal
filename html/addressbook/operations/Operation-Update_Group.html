Update group
Overview

The Update Group method enables updating an existing USER group identified by its Group ID and group ID is mandatory in a group.

If this UID is invalid or is a SYSTEM group’s UID, this API will return an InvalidGroupException. If group contains a non-empty name, this name will override the one server side if no already existing group has this name in subscriber’s address book. If a group already exists with this name, CSAB will return an InvalidGroupException (groups names are compared without taking into account the case).

If group contains an empty name, this API will return an InvalidGroupException.

If group does not contain a name, it will leave unchanged the name on the server side.

Return:

  • If successful, an empty envelope with an HTTP code of 204.
  • InvalidSubscriberException if the sub_id is incorrect or the subscriber’s account status is BLOCKED.
  • InvalidContactException if one of the contacts UID is not valid.
  • InvalidGroupException if the group UID is not valid.
  • SystemException for any other error.

Request Parameters
Parameter
authorization String Header

Specifies textquotedblBearertextquotedbl + OAuth Token – access_token. If the Authorization header is missing, the system shall return an HTTP 400 Invalid Request message. If the token is invalid the system shall return an HTTP 401 Unauthorized message with a WWW-Authenticate HTTP header.

content-type String Header

Specifies the representation format for the request. This is the content type for the whole message. The acceptable values for this parameter are:

  • application/json
  • application/xml
group Complex Body

Specifies the subscriber’s contact group object

groupId String URI

Specifies the group’s identifiers to be update.
