Get contact groups
Overview

The Get Contact Groups enables retrieving the list of groups a contact is belonging to. It takes a contact id as an argument and returns a list of groups. If contactId is invalid, this API will return an InvalidContactException.

If sortField contains invalid parameters, it is ignored. Default sorting is based on groups’ creation date, most recent first.

Return:

  • If the call is successful, it returns a list of group’s data structure.
  • InvalidContactException if the contact identifier is invalid.
  • InvalidSubscriberException if the sub_id is incorrect or the subscriber’s account status is BLOCKED.
  • SystemException for any other error.

Request Parameters
Parameter
accept String Header

Specifies the format of the body of the response. Valid values are:

  • application/json
  • application/xml

The default value is application/json. Per rfc2616: textquotedblIf no Accept header field is present, then it is assumed that the client accepts all media types.textquotedbl By default our services return application/json.

Normal Accept header processing rules shall be followed according to rfc2616.

accept-encoding String Header

Specifies accept encoding format

gzip,deflate

When this header is present, the gateway should compress the response to indicate the client that response is in compressed format

authorization String Header

Specifies textquotedblBearertextquotedbl + OAuth Token – access_token. If the Authorization header is missing, the system shall return an HTTP 400 Invalid Request message. If the token is invalid the system shall return an HTTP 401 Unauthorized message with a WWW-Authenticate HTTP header.

content-type String

Specifies the representation format for the request. This is the content type for the whole message. The acceptable values for this parameter are:

  • application/json
  • application/xml
contactId String URI

Specifies subscriber contact ID. Note : Should not use “me” in URI to retrieve My User Profile data model.
