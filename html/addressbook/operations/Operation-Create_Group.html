Create group
Overview

The Create Group method enables creating a USER group, if the subscriber’s address book did not reach its configurable maximum number of groups (configurable at the system level, 200 by default). Favorites group is not taken into account for this maximum number. The only mandatory field in group is a non-empty group name. If a group with the same name already exists on the server, this method will return an InvalidGroupException. The group names are case insensitive during the comparison.

If a group id (groupId) is present in group, it will be ignored. If a type different from USER is present in group, it will be ignored and reset to USER by default. This method returns the group id of the created group to be used for further operations.

Return:

  • If call is successful, then it returns a new group id (groupId) with the 201 OK.
  • InvalidSubscriberException if the subscriber id is incorrect or the subscriber’s account status is BLOCKED.
  • If group name already exists on the server, then it returns InvalidGroupExceptionName.
  • If any other error, then the SystemException will be returned.

Request Parameters
Parameter
authorization String Header

Specifies textquotedblBearertextquotedbl + OAuth Token – access_token. If the Authorization header is missing, the system shall return an HTTP 400 Invalid Request message. If the token is invalid the system shall return an HTTP 401 Unauthorized message with a WWW-Authenticate HTTP header.

content-type String

Specifies the representation format for the request. This is the content type for the whole message. The acceptable values for this parameter are:

  • application/json
  • application/xml
group Complex Body

Specifies ATT’s subscriber group data model
