Update contact
Overview

The Update Contact method updates a contact based on the provided contact data structure. The contact’s Id is a mandatory field in the request. Moreover, the same rules regarding the mandatory fields are applied as those described in section 3.3.1.1. For instance, if the request tries to remove firstName, lastName and company in the same call, the update will fail. The same rules regarding the picture size and format are applied. If the modifications lead to the creation of a contact duplicate, the update will fail and return an InvalidContactException.

The gender can’t be removed via this API as with the former AAB platform. If the gender is present in the request but empty, the value won’t be changed in database. Any fields encapsulating several items (phone, picture, email) will be considered as empty if there is no value (for instance, a phone with only types, picture with only encoding information, etc.), and ignored or removed, depending on the contact’s existing fields.

Returns:

  • If successful, an empty envelope with an HTTP code of 200.
  • InvalidSubscriberException if the sub_id is incorrect or the subscriber’s account status is BLOCKED.
  • InvalidContactException if the Id is not valid.
  • SystemException for any other error.

Request Parameters
Parameter
accept String Header

Specifies the format of the body of the response. Valid values are:

  • application/json
  • application/xml

The default value is application/json. Per rfc2616: textquotedblIf no Accept header field is present, then it is assumed that the client accepts all media types.textquotedbl By default our services return application/json.

Normal Accept header processing rules shall be followed according to rfc2616.

accept-encoding String Header

Specifies accept encoding format

gzip,deflate

When this header is present, the gateway should compress the response to indicate the client that response is in compressed format

authorization String Header

Specifies textquotedblBearertextquotedbl + OAuth Token – access_token. If the Authorization header is missing, the system shall return an HTTP 400 Invalid Request message. If the token is invalid the system shall return an HTTP 401 Unauthorized message with a WWW-Authenticate HTTP header.

content-type String Header

Specifies the representation format for the request. This is the content type for the whole message. The acceptable values for this parameter are:

  • application/json
  • application/xml
contact Complex Body

Specifies AT&T Mobile Subscriber’s Contact data structure. Ref Section 3.2.1

contactId String URI

Specifies subscriber contact ID.
