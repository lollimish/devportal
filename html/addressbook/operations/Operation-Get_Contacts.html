Get contacts
Overview

The Get Contact method enables retrieving all the contact information with some order and pagination criteria. If a valid pagination criterion is specified, only a contacts’ subset is returned. If pagination contains invalid parameters, this method will return an empty list.

Quick Contacts:

A Quick Contact is a limited set of popular fields in a contact, usually used for providing a better end user experience allowing a quick way of retrieving the contacts. AT&T Contact Services supports an operation to retrieve existing Contacts in the form of Quick Contacts using GET operation.

Return:

• If the call is successful, it returns a Contact ss list data structure containing all the contacts.

• InvalidSubscriberException if the sub_id is incorrect or the subscriber’s account status is BLOCKED.

• SystemException for any other error.

Request Parameters
Parameter
accept String Header

Specifies the format of the body of the response. Valid values are:

• application/json

• application/xml

The default value is application/json. Per rfc2616: \textquotedbl{}If no Accept header field is present, then it is assumed that the client accepts all media types.\textquotedbl{} By default our services return application/json.

Normal Accept header processing rules shall be followed according to rfc2616.

accept-encoding String Header

Specifies accept encoding format

gzip,deflate

When this header is present, the gateway should compress the response to indicate the client that response is in compressed format

authorization String Header

Specifies \textquotedbl{}Bearer\textquotedbl{} + OAuth Token – access\_token. If the Authorization header is missing, the system shall return an HTTP 400 Invalid Request message. If the token is invalid the system shall return an HTTP 401 Unauthorized message with a WWW-Authenticate HTTP header.

content-type String Header

Specifies the representation format for the request. This is the content type for the

\textbf

whole

message. The acceptable values for this parameter are:

• application/json

• application/xml

\multicolumn{5}{|l}{\textbfOutput filters}
x-fields String Header

Specifies String of comma \%2C (,) delimited field names that are expected on the response. These field names should match the Contact object. Ref 3.2.1.

Accepted options:

1)

\textbf

x-fields: shallow (OR)

shallow – Returns QuickContact Data model. If one or more search param are present in the request, then the x-fields value should set to shallow.

2)

\textbf

x-fields: {[}attribute names from

Contact data structure Ref 3.2.1 with comma separated {]}

\textbf

{[}OR{]

}

3)

\textbf

x-fields: (OR) no x-field header parameter

Returns all contacts will return all the attributes

Note:

a) Should use only one of the above options.

b) Combining options 1, 2, 3 are invalid. Return full contact object for this condition.

c) If a request contains search param and x-fields: {[}attribute names{]} or x-fields: or no x-fields header param, then default value for the x-fields is shallow {[}x-fields: shallow{]} should set in the request without returning any exception.

\multicolumn{5}{|l}{\textbfPagination Parameters – Invalid pagination parameter values should return empty list.}
order String Query String

Specifies the output sorting order. Valid values are

• ASC – ascending (Default)

• DESC – descending.

orderBy String Query String

Specifies the field name to be used for ordering the result set.

pageSize Integer Query String

Indicates page size. Must be equal to or greater than 1, if not specifies default as 10.

pageNo Integer Query String

Indicates page number. . Must be greater than 0, if not specifies default as 1.

\multicolumn{5}{|l}{\textbfSearch Parameters}
formattedName String Query String

Specifies formatted name and it is derived information.

\textbf

Personal type

: “prefix firstName middleName lastName suffix”

\textbf

Business type

: organization

firstName String Query String

Specifies first name of the contact. Mandatory if contact type is Personal

lastName String Query String

Specifies middle name of the contact

nickName String Query String

Specifies alternate name or familiar form of proper name

email String Query String

Specifies Email addresses associated with the contact such as official mail address, personal mail address etc.

phone String Query String

Specifies Phone numbers associated with the contacts, such as Personal phone number, office phone number. Can provide multiple phone numbers separated by a “\%2c” (,) to a maximum of 5 numbers.

organization String Query String

Specifies Organization name.

addressLine1 String Query String

Specifies Street Address Line 1

addressLine2 String Query String

Specifies Street Address Line 2

city String Query String

Specifies City

zipcode String Query String

Specifies State
