1. Introduction
Overview

AT&T Locker API gives authorized applications the ability to store,socialize and share user content that resides on users PC and Devices.This Interface Document (IA) gives on overview of Locker API, Payloadstructure, Errors returned, etc that will help a client developerbuild Locker client application. This document lists how to invokeAPI’s but doesn’t discuss orchestrating the API’s to make a businessfunction. The Getting Started Guide and Sequence Diagrams (not includedhere) will give a good overview of how to orchestrate these API’sto make a business function.

HTTP Method Override

If the calling client does not support HTTP DELETE, PATCH and/or PUT,the client may use the following header and call the API using HTTPPOST

x-http-method-override: DELETE

or

x-http-method-override: PATCH

or

x-http-method-override: PUT

XML Declaration

The XML documentation declaration shall be optional for requests thathave an XML body, but shall be required for responses that have anXML body.

Source Device IP Address

The APIGW shall retrieve the source device IP address and send thatwith every request to the Locker enabler set in the HTTP header \emph{device-remote-address}
