List all tracks
Overview

Lists all tracks belonging to a user.

  • OAuth Scope

  • Scope: LOCKER

  • Model: OAuth authorization_code

  • Resource
  • copy 
    /speechToText
Request Examples
  • Get a tracks listing

This demonstrates how to get a tracks listing.

copy 
GET /locker/v1/tracks?limit=20&offset=10&orderBy=author&order=desc HTTP/1.1 
host: api.att.com
authorization: Bearer abcdef12345678
accept: application/xml
accept-encoding: gzip

Request Parameters
Parameter
accept String Header

Specifies the format of the body of the response. Valid values are:

  • application/json

  • application/xml

The default value is application/json. Per rfc2616: \textquotedbl{}If no Accept header field is present, then it is assumed that the client accepts all media types.\textquotedbl{} By default our services return application/json.

Normal Accept header processing rules shall be followed according to rfc2616.


Note: If there is no entity body in a normal successful response, this parameter can still be specified to determine the format in the case of an error response message.

accept-encoding String Header

Specifies the accept encoding format. When this header is present, the gateway should compress the response to indicate the client that response is in compressed format. Valid values are: gzip

authorization String Header

Specifies the authorization type and token. \textquotedbl{}Bearer\textquotedbl{} + OAuth Token – access\_token. If the authorization header is missing, the system shall return an HTTP 400 Invalid Request message. If the token is invalid the system shall return an HTTP 401 Unauthorized message with a WWW-Authenticate HTTP header.

query String Query Parameter

URL encoded search query. We recommend using the UTF-8 encoding scheme to obtain the bytes for unsafe characters. This is in keeping with the following W3 recommendation: http://www.w3.org/TR/html40/appendix/notes.html\#non-ascii-chars

before String Query Parameter

Files added before or exactly at timestamp, if empty no lower time limit for search period, example: 2010-07-30

after String Query Parameter

Files added after or exactly at timestamp, if empty no lower time limit for search period, example: 2010-07-30

limit Integer Query Parameter

Specifies the max number of results returned.

offset Integer Query Parameter

Specifies the scroll set pointer on the results collection.

orderBy String Query Parameter

Specifies the sorting criteria. Valid values are: name, creationdate, contentsize, author

Default: name

order String Query Parameter

Specifies the sorting order. Valid values are: asc, desc

Default: asc

isCompressed Boolean Query Parameter

Specifies that the result set will only contain the name of the media instead of the full details if set to true. Otherwise default result set.

Require Parameters
Parameter
content-type String Header

Specifies the type of content of the body of the entity. Must be set to one of the following values:

  • applciation/json

  • application/xml

content-encoding String Header

Specifies the type of encoding used on the data. Supported values are: identity or gzip

content Object Body

Specifies the object that contains the response info.

total Integer Body

Specifies the total number of tracks in a user’s account

count Integer Body

Specifies the number of tracks returned in the response

tracks Object Body

Specifies the object that contains the tracks list.

track Object Body

Specifies the object that contains the tracks information

Av Object Parameters
Parameter
status String body

Specifies the status, and the options are: pending, error, ok, infected, riskware, spyware, adware

detection String body

Specifies the detection name if infected, missing otherwise

message String body

Specifies the message for specifying state in case not clean, key missing otherwise.
