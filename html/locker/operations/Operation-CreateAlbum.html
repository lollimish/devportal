Createalbum
Overview

The Create Album method is used to create a new album . The user is allowed to create an album with the name of an existing album.

  • OAuth Scope

  • Scope: LOCKER

  • Model: OAuth authorization_code

  • Resource
  • copy 
    /speechToText
Request Examples
  • XML
  • JSON

This demonstrates how to create an album. The response format will be in XML.

copy 
POST /locker/v1/albums HTTP/1.1 
Host: api.att.com
authorization: bearer abcdef12345678
accept: application/xml
content-type: application/xml

<album>
      <name>somealbum</name>
      <description>Old School</description>
</album>

This demonstrates how to create an album. The response format will be in JSON.

copy 
POST /locker/v1/albums HTTP/1.1 
Host: api.att.com 
authorization: bearer abcdef12345678
accept: application/json
content-type: application/json

{
  "album": {
    "name": "somealbum",
    "description": "Old School"
  }
}

Request Parameters
Parameter
accept String Header

Specifies the format of the body of the response. Valid values are:

  • application/json

  • application/xml

The default value is application/json. Per rfc2616: "If no Accept header field is present, then it is assumed that the client accepts all media types." By default our services return application/json.

Normal Accept header processing rules shall be followed according to rfc2616.


Note: If there is no entity body in a normal successful response, this parameter can still be specified to determine the format in the case of an error response message.

accept-encoding String Header

Specifies the accept encoding format. When this header is present, the gateway should compress the response to indicate the client that response is in compressed format. Valid values are: gzip

authorization String Header

Specifies the authorization type and token. "Bearer" + OAuth Token â€“ access_token. If the authorization header is missing, the system shall return an HTTP 400 Invalid Request message. If the token is invalid the system shall return an HTTP 401 Unauthorized message with a WWW-Authenticate HTTP header.

content-length Integer Header

Specifies the length of the content in octets.

content-type String Header

Specifies the type of content of the body of the entity. Must be set to one of the following values:

  • application/json

  • application/xml

album Object Body

Contains the parameters that define the new album

name String Body

Specifies the name of album

description String Body

Specifies the description of the album
