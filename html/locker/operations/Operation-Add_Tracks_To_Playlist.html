Add tracks to playlist
Overview

Insert a track into a playlist belonging to a user. Adding the sametrack twice in the same playlist is allowed. Considering the followingscenario: A playlist “Antarctica” contains Song1.mp3

If a user selects 3 tracks, Song1.mp3, Song2.mp3, Song3.mp3 and addsthem to the playlist “Antarctica”. Playlist “Antarctica” will contain,the following: Song1.mp3, Song1.mp3, Song2.mp3 and Song3.mp3.

  • OAuth Scope

  • Scope: LOCKER

  • Model: OAuth authorization_code

Request Examples
  • Add tracks to an playlist in XML
  • Add tracks to an playlist in JSON

This demonstrates how to add tracks to an playlist, and specifying the request to be in XML.

copy 
POST /locker/v1/playlists/3433435/tracks HTTP/1.1 
host: api.att.com
content-type: application/xml
authorization: Bearer abcdef12345678
accept: application/xml

<trackIds>
    <id>112</id>
    <id>125</id>
    <index>1</index>  	
</trackIds>

This demonstrates how to add tracks to an playlist, and specifying the request to be in JSON.

copy 
POST /locker/v1/folders/3433435/files HTTP/1.1 
Host: api.att.com
content-type: application/json
authorization: Bearer abcdef12345678
accept: application/json

{
  "trackIds": {
    "id": [112,125],
    "index": 1
  }
}

Request Parameters
Parameter
accept String Header

Specifies the format of the body of the response. Valid values are:

  • application/json

  • application/xml

The default value is application/json. Per rfc2616: \textquotedbl{}If no Accept header field is present, then it is assumed that the client accepts all tracks types.\textquotedbl{} By default our services return application/json.

Normal Accept header processing rules shall be followed according to rfc2616.


Note: If there is no entity body in a normal successful response, this parameter can still be specified to determine the format in the case of an error response message.

accept-encoding String Header

Specifies the accept encoding format. When this header is present, the gateway should compress the response to indicate the client that response is in compressed format. Valid values are: gzip

authorization String Header

Specifies the authorization type and token. \textquotedbl{}Bearer\textquotedbl{} + OAuth Token – access\_token. If the authorization header is missing, the system shall return an HTTP 400 Invalid Request message. If the token is invalid the system shall return an HTTP 401 Unauthorized message with a WWW-Authenticate HTTP header.

content-length Integer Header

Specifies the length of the content in octets. This header parameter is only required for the non-streaming request.

content-type String Header

Specifies the type of content of the body of the entity. Must be set to one of the following values:

  • application/json

  • application/xml

playlistId Integer URI Path

Specifies the id of the playlist where the files will be placed.

trackIds Object Body

Contains the tracks that are to be added

id Array Body

Specifies the ids of the track that is to be added to the playlist.

index Integer Body

Specifies the absolute position where you want to place the new content.

Default is to put at the end of the playlist.
