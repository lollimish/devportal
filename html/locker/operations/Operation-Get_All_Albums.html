Get all albums
Overview

Request Examples
  • Get albums based on query parameters

This demonstrates how to get albums using GET, and specifying the response to be in XML.

copy 
GET /locker/v1/albums?limit=5&offset=5&orderBy=name&order=asc HTTP/1.1 
host: api.att.com
authorization: Bearer abcdef12345678
accept: application/xml

Request Parameters
Parameter
accept String Header

Specifies the format of the body of the response. Valid values are:

  • application/json

  • application/xml

The default value is application/json. Per rfc2616: "If no Accept header field is present, then it is assumed that the client accepts all media types." By default our services return application/json.

Normal Accept header processing rules shall be followed according to rfc2616.


Note: If there is no entity body in a normal successful response, this parameter can still be specified to determine the format in the case of an error response message.

accept-encoding String Header

Specifies the accept encoding format. When this header is present, the gateway should compress the response to indicate the client that response is in compressed format. Valid values are: gzip

authorization String Header

Specifies the authorization type and token. "Bearer" + OAuth Token – access_token. If the authorization header is missing, the system shall return an HTTP 400 Invalid Request message. If the token is invalid the system shall return an HTTP 401 Unauthorized message with a WWW-Authenticate HTTP header.

limit% {} Integer Query Parameter

Specifies the max number of results returned. Valid value range is 1 to 100.

% \begin{comment} Made it required, so no more default \end{comment} {}
offset% {} Integer Query Parameter

Specifies the scroll set pointer on the results collection.

orderBy String Query Parameter

Specifies the sorting criteria. The valid values for sort are: name, creationDate, size, count

% {\footnotesize \par}

Default: name

order String Query Parameter

The sorting order. The valid values are: asc, desc

Default: asc

% {}
Require Parameters
Parameter
content-encoding String Header

Specifies the type of encoding used on the data. Supported values are: identity or gzip

content-type String Header

Specifies the type of content of the body of the entity. Must be set to one of the following values:

  • application/xml

  • application/json


Note: If there is no entity body in a normal successful response, this parameter is still needed to specify the format in the case of an error response message.

albums Object Body

Contains the album objects

total Integer Body

Specifies the total number of albums in a user’s account

count Integer Body

Specifies the number of albums returned in this response

album Object Body

Contains the object which has the album information
